What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. Continuous Integration means developers frequently push code to a repository where automatic checks and builds are executed. Continuous Deployment means that after successful checks, the application is automatically deployed. This process reduces manual work and helps detect errors early. CI/CD is widely used in modern cloud systems to automate software delivery.

My Pipeline

When I push my code to GitHub, GitHub Actions automatically starts the workflow. It checks that the required files exist and then deploys the website to GitHub Pages. After deployment finishes, the live website is updated automatically.
